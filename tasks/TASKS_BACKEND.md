# Agent: Backend

**Can start:** After `packages/types` and `packages/utils` are scaffolded.
**Owns:** Everything under `apps/api/`.
**Critical output:** `apps/api/openapi.json` — the API contract that Web and Mobile
agents depend on. Publish this as early as possible, even before all endpoints
are fully implemented.

Stack: Go · Echo v4 · Bun · PostgreSQL

---

## 3. Auth Handlers (`/api/auth`)

### JWT Middleware

- [ ] `middleware/jwt.go` — Echo middleware that:
  - Extracts Bearer token from `Authorization` header
  - Validates and parses JWT using `golang-jwt/jwt`
  - Stores parsed claims in Echo context (`c.Set("user", claims)`)
- [ ] Public route group: `/api/auth/*` — no JWT middleware
- [ ] Protected route group: all other `/api/*` routes — JWT middleware applied

### Endpoints

- [ ] `POST /api/auth/register`
  - Body: `{ email, username, displayName, password }`
  - Validate unique email and username (query DB)
  - Hash password with `bcrypt.GenerateFromPassword` (cost 12)
  - Generate cuid for user ID
  - Insert user via Bun
  - Send welcome email asynchronously (goroutine)
  - Issue access + refresh tokens, return `{ accessToken, refreshToken, user }`

- [ ] `POST /api/auth/login`
  - Body: `{ email, password }`
  - Fetch user by email, `bcrypt.CompareHashAndPassword`
  - Issue tokens, return `{ accessToken, refreshToken, user }`

- [ ] `POST /api/auth/refresh`
  - Body: `{ refreshToken }`
  - Hash incoming token, look up in `RefreshToken` table
  - Check not expired
  - Delete old token, insert new token (rotation)
  - Return new `{ accessToken, refreshToken }`

- [ ] `POST /api/auth/logout`
  - Body: `{ refreshToken }`
  - Delete matching `RefreshToken` row

- [ ] `POST /api/auth/forgot-password`
  - Body: `{ email }`
  - Generate secure random token, hash it, store with 15min expiry in a `PasswordResetToken` table
  - Send reset email via go-mail

- [ ] `POST /api/auth/reset-password`
  - Body: `{ token, newPassword }`
  - Validate token exists and is not expired
  - Update password hash, delete reset token, delete all refresh tokens for user

## 6. Groups Handlers (`/api/groups`)

### Group CRUD

- [ ] `GET /api/groups` — list groups for current user with net balance per group
- [ ] `POST /api/groups` — create group
  - Body: `{ name, imageUrl?, startDate?, endDate?, maxMembers?, defaultSplit?, simplifyDebts? }`
  - Generate `invite_code` (UUID)
  - Insert group + insert GroupMember row as OWNER in a transaction
- [ ] `GET /api/groups/:id` — group detail + members + balances
- [ ] `PATCH /api/groups/:id` — update group fields (owner only)
- [ ] `DELETE /api/groups/:id` — delete group and cascade (owner only)

### Member Management

- [ ] `POST /api/groups/:id/members` — add member by `{ userId }` or `{ email }` or `{ username }`
  - Enforce `max_members` limit
- [ ] `DELETE /api/groups/:id/members/:userId` — remove member (owner only)
- [ ] `POST /api/groups/:id/leave` — current user leaves group
- [ ] `GET /api/groups/:id/members` — list members with per-member balance

### Invite Links

- [ ] `POST /api/groups/:id/invite/regenerate` — replace `invite_code` with new UUID (owner only)
- [ ] `GET /api/groups/join/:inviteCode` — public preview (no auth required)
- [ ] `POST /api/groups/join/:inviteCode` — join group (auth required), enforces `max_members`

### Balances

- [ ] `GET /api/groups/:id/balances`
  - Query all expenses + splits + settlements for the group
  - Call balance calculation logic in `internal/services/balance.go`
  - If `simplify_debts` is true: run simplification algorithm
  - Return per-member balance array
